## 2. บทสรุปผู้บริหาร (Executive Summary)

บริษัท ศาลาแดง จำกัด ("SLD") มีความยินดีนำเสนอขอบเขตการทำงานสำหรับโครงการวางระบบ {{SAP Business One / Oracle NetSuite}} ให้กับ {{ชื่อบริษัทลูกค้า}} ("ลูกค้า") โดยโครงการนี้มีวัตถุประสงค์เพื่อ{{สรุปเป้าหมายหลัก เช่น ยกระดับระบบบริหารจัดการทรัพยากรองค์กรให้ครบวงจร ครอบคลุมงานบัญชี-การเงิน จัดซื้อ ขาย และคลังสินค้า}}

ระบบ ERP จะช่วยให้ลูกค้าสามารถ{{ระบุ value proposition หลัก เช่น ลดขั้นตอนงานซ้ำซ้อน เพิ่มความถูกต้องของข้อมูลทางการเงิน และรองรับการขยายธุรกิจในอนาคต}} ทั้งนี้ SLD จะดำเนินการตามมาตรฐาน implementation methodology ที่ผ่านการพิสูจน์จากโครงการกว่า 50 องค์กรในประเทศไทยและภูมิภาค

### 4.2 ขอบเขตที่ไม่รวมในโครงการ (Out of Scope)

> สิ่งต่อไปนี้ **ไม่** รวมอยู่ในโครงการนี้ หากต้องการเพิ่มเติมจะดำเนินการผ่านกระบวนการ Change Request

1. ไม่รวมข้อมูลรายการค้า (Transaction Data) และการบันทึกข้อมูลเก่า (Historical Data Entry) จากระบบเดิม
2. ไม่รวมการพัฒนาโปรแกรมเพิ่มเติม (Program Customization / Add-On) และการเชื่อมต่อระบบภายนอก (Integration) นอกจากที่ระบุไว้ใน Scope
3. ไม่รวมยอดยกมา (Opening Balance) {{ระบุ: รวม/ไม่รวม — ถ้ารวมให้ย้ายไป In-Scope}}
4. ไม่รวมโมดูลที่ไม่ได้ระบุข้างต้น เช่น {{ระบุ Module ที่ไม่รวม เช่น CRM, Manufacturing, Production, HR/Payroll}}
5. ไม่รวมฮาร์ดแวร์ เซิร์ฟเวอร์ และค่าลิขสิทธิ์ซอฟต์แวร์ (Hardware / Server / License)
6. ไม่รวมการสนับสนุนหลังสิ้นสุดระยะ Support Period ที่กำหนด
7. ไม่รวมรายงานพิเศษที่ต้องพัฒนาเพิ่มเติมนอกเหนือจาก {{จำนวนรายงาน}} รายงานที่ระบุไว้
8. ไม่รวมการแก้ไขข้อผิดพลาดที่เกิดจากซอฟต์แวร์ของบุคคลที่สาม (Third-Party) หรือการใช้งานที่ไม่ถูกต้อง

## 8. สมมติฐานโครงการ (Project Assumptions)

> สมมติฐานต่อไปนี้เป็นพื้นฐานในการประเมินขอบเขตงานและค่าใช้จ่าย หากมีการเปลี่ยนแปลงจากสมมติฐานเหล่านี้ อาจส่งผลต่อระยะเวลาและค่าใช้จ่ายของโครงการ

1. **ระยะเวลา SOW (SOW Term):** บริการตาม SOW ฉบับนี้มีผลภายใน 12 เดือนนับจากวันที่ออกใบแจ้งหนี้ หากเกินกำหนดต้องตกลงเป็นลายลักษณ์อักษร
2. **ข้อจำกัดขอบเขต (Scope Limitation):** ขอบเขตงานจำกัดเฉพาะรายการใน Section 4.1 เท่านั้น งานนอกขอบเขตจะดำเนินการผ่าน Change Request และอาจมีค่าใช้จ่ายเพิ่มเติม
3. **การให้บริการ (Service Delivery):** บริการทั้งหมดจะดำเนินการแบบทางไกล (Remote) ผ่าน Video Conference, อีเมล หรือระบบสื่อสารออนไลน์ {{ยกเว้นกรณี Onsite — ดู clause ค่าเดินทาง}}
4. **ความพร้อมทรัพยากรลูกค้า (Customer Resource Availability):** ลูกค้าจัดเตรียมบุคลากรหลัก (Key User) และผู้มีอำนาจตัดสินใจเข้าร่วมกิจกรรมโครงการตามกำหนดการ ความล่าช้าจากฝั่งลูกค้าอาจส่งผลต่อ timeline และค่าใช้จ่าย
5. **คุณภาพข้อมูล (Data Quality):** ลูกค้าเป็นผู้รับผิดชอบความถูกต้องและครบถ้วนของข้อมูลหลัก (Master Data) ที่จัดเตรียมสำหรับการย้ายข้อมูล SLD จะไม่รับผิดชอบข้อผิดพลาดที่เกิดจากข้อมูลต้นทางที่ไม่ถูกต้อง
6. **การยกเลิก/เลื่อนนัด (Cancellation):** ทั้งสองฝ่ายจะใช้ความพยายามตามสมควรเพื่อเข้าร่วมประชุมตามกำหนด การยกเลิกซ้ำโดยไม่แจ้งล่วงหน้าอาจส่งผลให้เกิดความล่าช้าและค่าใช้จ่ายเพิ่มเติม
7. **รูปแบบรายงาน (Report Templates):** รายงานและแบบฟอร์มจะใช้ Template มาตรฐานของ Sala Daeng การปรับแต่งนอกเหนือจากนี้ถือเป็นงานนอกขอบเขต
8. **จำนวนฐานข้อมูล (Database Scope):** ขอบเขตจำกัดเฉพาะ {{ระบุจำนวน เช่น 1}} ฐานข้อมูลจริง (Production) และ {{ระบุจำนวน เช่น 1}} ฐานข้อมูลทดสอบ (Test) หากต้องการเพิ่มต้องจัดทำ SOW แยก
9. **ปริมาณการย้ายข้อมูล (Data Volume):** หากปริมาณข้อมูลเกิน limit ที่ระบุไว้ใน Section 4.1 SLD สงวนสิทธิ์ในการออก Change Order สำหรับงานเพิ่มเติม
10. **ชำระเงิน (Payment):** ลูกค้าตกลงชำระค่าบริการก่อนเริ่มโครงการ (Project Kick-Off) ตามเงื่อนไขที่ระบุใน Section 10

## 9. การบริหารการเปลี่ยนแปลง (Change Management)

### กระบวนการ Change Request

หากมีความต้องการเพิ่มเติมนอกเหนือจากขอบเขตที่ระบุไว้ จะดำเนินการตามขั้นตอนดังนี้:

| ขั้นตอน | รายละเอียด | ผู้รับผิดชอบ |
|---|---|---|
| 1. ยื่นคำขอ | ลูกค้าแจ้งความต้องการเพิ่มเติมเป็นลายลักษณ์อักษร (อีเมลหรือแบบฟอร์ม Change Request) | ลูกค้า |
| 2. ประเมินผลกระทบ | SLD ประเมินผลกระทบต่อระยะเวลา ขอบเขต และค่าใช้จ่าย แจ้งลูกค้าภายใน 5 วันทำงาน | SLD |
| 3. อนุมัติ | ทั้งสองฝ่ายลงนามอนุมัติ Change Order ก่อนเริ่มดำเนินการ | ทั้งสองฝ่าย |
| 4. ดำเนินการ | SLD ดำเนินงานตาม Change Order ที่อนุมัติแล้ว | SLD |

> **สำคัญ:** SLD จะ **ไม่** ดำเนินงานนอกขอบเขตใดๆ หากไม่ได้รับอนุมัติ Change Order เป็นลายลักษณ์อักษรจากทั้งสองฝ่าย

### เงื่อนไขที่อาจนำไปสู่ Change Order:
- ลูกค้าต้องการงานที่ไม่ได้ระบุไว้ใน SOW ฉบับนี้
- จำนวนข้อมูล (Records) เกิน limit ที่ระบุไว้
- มีการเปลี่ยนแปลงกระบวนการทางธุรกิจระหว่างดำเนินโครงการ
- ต้องการรายงานเพิ่มเติมนอกจากที่ระบุไว้
- ต้องการ Module หรือ Integration เพิ่มเติม
