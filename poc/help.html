ภาพรวม

ดูงบแต่ละโปรเจค: ตั้งไว้เท่าไหร่ ใช้ไปเท่าไหร่ เหลือเท่าไหร่ แยกตามประเภท

โครงสร้างงบ

  • รายการงบ (Budget Line) — เช่น "อุปกรณ์ Radar ฿350,000"
  • จัดสรร (Allocated) — งบที่วางแผนไว้
  • ใช้ไป (Used) — ยอดจริงจาก PR/PO/Expense
  • คงเหลือ (Remaining) — ยอดที่เหลือใช้ได้

วิธีใช้งาน

  • เลือกโปรเจคเพื่อดูงบ
  • กด + เพิ่มรายการงบ เพื่อเพิ่มรายการใหม่
  • คลิกที่ตัวเลขเพื่อแก้ไขจำนวนเงิน
  • ดู % การใช้งบและคำเตือนเกินงบ
💡 พอ PR/PO ได้รับอนุมัติ งบจะถูกหักอัตโนมัติ ไม่ต้องกรอกเอง
การเงินราย Phase (Phase Finance) เอกสาร ▼

ภาพรวม

ดูรายรับ ต้นทุน และกำไรของโปรเจคแยกตาม Phase ได้ชัดเจน ช่วยให้รู้ว่า Phase ไหนกำไร Phase ไหนขาดทุน

a) งวดรับเงิน (Payment Schedule)

กำหนดว่าแต่ละ Phase ลูกค้าจ่ายเงินเมื่อไหร่ เท่าไหร่

  • เข้าหน้า Plan Project แล้วกดแท็บ Payment Schedule
  • กด + เพิ่มงวด ระบุ Phase, จำนวนเงิน, วันครบกำหนด
  • เมื่อรับเงินแล้วกดเปลี่ยนสถานะเป็น "รับแล้ว"
  • ดูสรุปงวดค้างรับและงวดที่ใกล้ครบกำหนด
กำหนดงวด→ ส่งงาน→ วางบิล→ รับเงิน

b) ต้นทุนราย Phase

ดูต้นทุนที่เกิดขึ้นจริงในแต่ละ Phase โดยดึงจาก PR/PO ที่ลิงก์กับ Phase นั้น

  • เข้าแท็บ Phase Cost เลือก Phase ที่ต้องการ
  • ดูรายการ PR/PO ที่ผูกกับ Phase พร้อมยอดเงิน
  • กดลิงก์เลขเอกสารเพื่อเปิด PR/PO ต้นทาง
  • ดู % ต้นทุนเทียบกับงบที่ตั้งไว้

c) ภาพรวมกำไร-ขาดทุน

สรุปรายรับ - ต้นทุน = กำไร ของแต่ละ Phase และภาพรวมทั้งโปรเจค

คอลัมน์คำอธิบาย
รายรับ (Revenue)ยอดจาก Payment Schedule ที่รับแล้ว
ต้นทุน (Cost)ยอดจาก PR/PO/Expense ที่ผูกกับ Phase
กำไร (Profit)รายรับ - ต้นทุน (สีเขียว = กำไร, สีแดง = ขาดทุน)
Margin %กำไร / รายรับ x 100
💡 ดู Margin % ติดลบแปลว่า Phase นั้นขาดทุน ต้องทบทวนต้นทุนด่วน
ใบขอซื้อ / ใบสั่งซื้อ (PR / PO) เอกสาร ▼

ภาพรวม

สร้างใบขอซื้อ (PR) แล้วต่อเป็นใบสั่งซื้อ (PO) ดูแบบบอร์ดหรือตารางก็ได้

Document Flow

PR (ใบขอซื้อ)→ PO (ใบสั่งซื้อ)→ ส่งเข้า SAP→ เลขเอกสาร SAP

สถานะเอกสาร

Draft (ร่าง)→ Pending (รออนุมัติ)→ Approved (อนุมัติ)→ Sent to SAP→ Received

วิธีใช้งาน

  • กด + New PR หรือ + New PO สร้างเอกสาร
  • กรอก: โปรเจค, ผู้ขาย, รายการสินค้า (สินค้า, จำนวน, ราคา)
  • สลับมุมมอง Kanban / List ได้
  • คลิกเลขเอกสารเพื่อดูรายละเอียดและ Document Flow
💡 พอ SAP รับเอกสาร จะได้เลขที่จริงกลับมาเป็น badge สีฟ้าในระบบ
เงินทดรองจ่าย (Advance / Reimburse) เอกสาร ▼

ภาพรวม

เงินทดรองมี 2 ประเภท: เบิกล่วงหน้า (บริษัทจ่ายก่อน) และเบิกคืน (พนักงานจ่ายก่อน) เลือกประเภทตอนสร้างเอกสาร

เบิกล่วงหน้า (Advance)

บริษัทจ่ายเงินให้พนักงานก่อน พนักงานไปใช้จ่ายแล้วกลับมาเคลียร์ภายใน 7 วัน

  • สร้างเอกสารเลือกประเภท เบิกล่วงหน้า
  • ส่งอนุมัติ รอหัวหน้า/การเงินอนุมัติตาม tier
  • การเงินจ่ายเงินให้พนักงาน แนบสลิป
  • พนักงานเคลียร์ใบเสร็จภายใน 7 วัน (ดู countdown badge)
  • การเงินตรวจใบเสร็จแล้วกดเคลียร์
ฉบับร่าง→ รออนุมัติ→ อนุมัติแล้ว→ จ่ายแล้ว→ เคลียร์→ เคลียร์แล้ว
ถ้าเกิน 7 วันไม่เคลียร์ badge จะเปลี่ยนเป็น "เกินกำหนด" และระบบล็อคไม่ให้สร้างเอกสารเบิกใหม่จนกว่าจะเคลียร์

เบิกคืน (Reimburse)

พนักงานจ่ายเงินตัวเองไปก่อน แล้วส่งใบเสร็จขอเบิกคืนจากบริษัท ไม่ต้องเคลียร์

  • สร้างเอกสารเลือกประเภท เบิกคืน
  • แนบใบเสร็จพร้อมยอดเงินที่จ่ายไป
  • ส่งอนุมัติ รอหัวหน้า/การเงินอนุมัติ
  • การเงินจ่ายเงินคืนให้พนักงาน จบ
ฉบับร่าง→ รออนุมัติ→ อนุมัติแล้ว→ จ่ายคืนแล้ว

ระดับการอนุมัติ

มากกว่า 100,000 บาทผู้จัดการ + การเงิน + ผู้บริหาร

เงินสดย่อย (Petty Cash) เอกสาร ▼

ภาพรวม

เงินสดในออฟฟิศสำหรับจ่ายค่าเบ็ดเตล็ด เช่น แท็กซี่ กาแฟ พัสดุ ระบบดูแลยอดและเตือนเมื่อเงินใกล้หมด

ขั้นตอน

สร้างกองทุน→ เบิกจ่าย→ ส่งใบเสร็จ→ อนุมัติ→ เติมเงิน→ นับเงินปิดงวด

วิธีใช้งาน

  • สร้างกองทุน — ตั้งชื่อ กำหนดวงเงิน เลือกคนดูแล
  • เบิกจ่าย — จ่ายเงินสด ลงบันทึกว่าจ่ายใคร เท่าไหร่
  • ส่งใบเสร็จ — พนักงานเอาใบเสร็จมาลง ระบบคำนวณเงินทอน
  • อนุมัติ — คนดูแล/การเงินเช็คใบเสร็จแล้วกดอนุมัติ
  • เติมเงิน — เงินใกล้หมดก็กดขอเติม
  • นับเงินปิดงวด — นับเงินสดจริง ระบบเทียบให้ว่าตรงไหม

ยานพาหนะ (Vehicle) ทรัพยากร ▼

ภาพรวม

จัดการรถบริษัทครบวงจร: ข้อมูลรถ ประกัน เคลม ซ่อมบำรุง จองรถ และแจ้งเตือนอัตโนมัติ

a) ข้อมูลรถ + สัญญาเช่า

ดูข้อมูลรถแต่ละคัน: ยี่ห้อ รุ่น สี ทะเบียน เลขตัวถัง สัญญาเช่า

  • กดที่การ์ดรถเพื่อเปิด modal รายละเอียด
  • ดูข้อมูลทั่วไป สัญญาเช่า วันหมดอายุ
  • กดปุ่มแก้ไขเพื่ออัปเดตข้อมูล (เฉพาะ Manager/Finance/Executive)

b) ประกันรถ

จัดการกรมธรรม์ประกันของรถแต่ละคัน

  • กดแท็บ Insurance ในหน้ารายละเอียดรถ
  • กด + เพิ่มกรมธรรม์ กรอกข้อมูลประกัน
  • Badge สี: เขียว = ยังไม่หมดอายุ, เหลือง = ใกล้หมด (30 วัน), แดง = หมดแล้ว

c) แจ้งเคลม

เกิดอุบัติเหตุหรือเสียหาย แจ้งเคลมผ่านระบบได้เลย

  • กด + แจ้งเคลม กรอกรายละเอียดเหตุการณ์
  • กรอกรายละเอียดความเสียหาย มูลค่า ชื่อผู้ขับ
  • ติดตามสถานะเคลมได้จากหน้ารายละเอียดรถ

d) ซ่อมบำรุง + แจ้งปัญหา

บันทึกประวัติซ่อมบำรุงและแจ้งปัญหาตอน check-in รถ

  • กดแท็บ Maintenance ดูประวัติซ่อมทั้งหมด
  • กด + บันทึกซ่อม ลงวันที่ รายการ ค่าใช้จ่าย
  • ตอน check-in รถ แจ้งปัญหาที่พบได้ทันที

e) แจ้งเตือนอัตโนมัติ

ระบบแจ้งเตือนเรื่องสำคัญให้อัตโนมัติ ไม่ต้องจำเอง

ประเภทแจ้งเตือนเมื่อ
ประกันรถใกล้หมดอายุ 30 วัน
ทะเบียนรถ (พ.ร.บ.)ใกล้หมดอายุ 30 วัน
สัญญาเช่าใกล้หมดอายุ 60 วัน
ซ่อมบำรุง (เลขไมล์)ใกล้ถึงรอบเช็คระยะ
ซ่อมบำรุง (วันที่)ถึงกำหนดเช็คตามรอบเวลา

f) ตารางรถว่าง

ดูปฏิทินว่ารถคันไหนว่างวันไหน จองจากปฏิทินได้เลย

  • เข้าแท็บ Calendar ดูตารางรถทั้งหมด
  • สีเขียว = ว่าง, สีฟ้า = จองแล้ว, สีเทา = ซ่อมบำรุง
  • คลิกช่องว่างบนปฏิทินเพื่อจองรถ

การคำนวณราคาเสนอ

  • Total Cost = วัสดุ + แรงงาน + ขนส่ง + อื่นๆ
  • Margin % = กำไรที่ต้องการ (กำหนดเป็น %)
  • Margin Amount = Total Cost x Margin %
  • Bid Price = Total Cost + Margin Amount

ใบเสนอราคา (Sales Quotation) เอกสาร ▼

ภาพรวม

สร้างใบเสนอราคาให้ลูกค้า ระบุรายการสินค้า/บริการ ส่วนลด VAT ส่งลูกค้ารอตอบรับ หรือสร้างจาก Booking ก็ได้

สถานะเอกสาร

Draft (ร่าง)→ Sent (ส่งลูกค้า)→ Accepted (ยอมรับ)/ Rejected (ปฏิเสธ)

วิธีใช้งาน

  • กด + สร้างใบเสนอราคา จากหน้ารายการ
  • กรอกข้อมูลลูกค้า ไซต์ ขนาดระบบ ประเภทหลังคา
  • เพิ่มรายการสินค้าจาก Item Master (ค้นหาได้ทั้งรหัสและชื่อ)
  • ระบุส่วนลด % ระบบคำนวณ VAT 7% ให้อัตโนมัติ
  • กด Save บันทึก → กด Send to Customer ส่งลูกค้า
  • ลูกค้าตอบรับ → กด Customer Accepted
  • กด Print พิมพ์ใบเสนอราคาได้ทุกเมื่อ
  • กด Copy to New คัดลอกเป็นใบเสนอราคาใหม่ (เช่น แก้ราคาแล้วส่งใหม่)

สร้างจาก Booking

ถ้ามี Booking (นัดสำรวจ/ติดตั้ง) ที่ยังไม่ได้สร้าง SQ ระบบจะแสดงใน Dashboard เป็น "รอสร้างใบเสนอราคา" กด Convert ได้เลย ข้อมูลลูกค้าและไซต์จะถูกดึงมาให้อัตโนมัติ

Document Navigation

ในหน้ารายละเอียด SQ กดปุ่ม « » ด้านบนเพื่อเลื่อนดูเอกสารตัวอื่นได้ทันที ไม่ต้องกลับหน้ารายการ

💡 เลขที่ SQ ถูกสร้างอัตโนมัติ (SQ-YYYYMM-001) ไม่ต้องกรอกเอง

ภาพรวม

ระบบจองทรัพยากร 3 ประเภท แสดงเป็นปฏิทินแบบ Google Calendar ดูว่าทรัพยากรไหนว่าง/ไม่ว่างได้ทันที

ประเภทการจอง

ประเภทคำอธิบายข้อมูลที่ต้องกรอก
รถ (Vehicle)จองรถบริษัทไปทำงานเลือกรถ, วันที่, โปรเจค
ช่างเทคนิค (Technician)จองช่างไปติดตั้ง/ซ่อมเลือกช่าง, วันที่, ไซต์งาน
ตั๋วเครื่องบิน (Flight)จองเที่ยวบินให้ทีมงานเที่ยวบิน, สนามบินต้นทาง/ปลายทาง

มุมมองปฏิทิน

  • Day — ดูทีละวัน แสดง booking เรียงตามเวลา
  • Week — ดูทั้งสัปดาห์ แต่ละคอลัมน์คือ 1 วัน
  • Month — ดูภาพรวมทั้งเดือน

วิธีใช้งาน

  • กดปุ่ม + จอง ที่ Sidebar ซ้าย เลือกประเภท
  • กรอกข้อมูล เลือกทรัพยากร วันที่ โปรเจค
  • ระบบเช็คว่าทรัพยากรว่างให้อัตโนมัติ (Availability Check)
  • ใช้ Sidebar Filter กรองดู booking ตามประเภท/สถานะ
  • Drag-to-Copy — ลาก booking ที่มีอยู่ไปวางวันอื่นเพื่อคัดลอก

สถานะการจอง

Pending→ Approved→ Checked Out→ Checked In
