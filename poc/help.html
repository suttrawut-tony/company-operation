ภาพรวม

ดูว่าแต่ละโปรเจคตั้งงบไว้เท่าไหร่ ใช้ไปเท่าไหร่แล้ว เหลือเท่าไหร่ แยกตามประเภทให้ดูง่าย

โครงสร้างงบประมาณ

  • รายการงบ (Budget Line) — รายการงบแต่ละตัว เช่น "อุปกรณ์ Radar ฿350,000"
  • จัดสรร (Allocated) — งบที่วางแผนไว้
  • ใช้ไป (Used) — ยอดที่ใช้จริง (จาก PR/PO/Expense)
  • คงเหลือ (Remaining) — ยอดที่ยังเหลือใช้ได้

วิธีใช้งาน

  • เลือกโปรเจคเพื่อดูงบประมาณ
  • คลิก + เพิ่มรายการงบ เพื่อเพิ่มรายการใหม่
  • แก้ไขจำนวนเงินได้โดยคลิกที่ตัวเลข
  • ดูเปอร์เซ็นต์การใช้งบและคำเตือนเกินงบ
พอ PR/PO ได้รับอนุมัติ งบจะถูกหักให้อัตโนมัติ ไม่ต้องมานั่งกรอกเอง
ใบขอซื้อ / ใบสั่งซื้อ (PR / PO) เอกสาร ▼

ภาพรวม

ใช้สร้างใบขอซื้อ (PR) แล้วต่อเป็นใบสั่งซื้อ (PO) ได้ ดูแบบบอร์ดหรือตารางก็ได้ตามสะดวก

ขั้นตอนเอกสาร (Document Flow)

PR (ใบขอซื้อ)→ PO (ใบสั่งซื้อ)→ ส่งเข้า SAP→ เลขเอกสาร SAP

สถานะเอกสาร

สถานะคำอธิบาย
Draft (ร่าง)สร้างแล้ว ยังไม่ส่งอนุมัติ
Pending Approval (รออนุมัติ)ส่งแล้ว รอผู้จัดการ/การเงินอนุมัติ
Approved (อนุมัติแล้ว)อนุมัติแล้ว พร้อมส่งเข้า SAP
Sent to SAP (ส่งเข้า SAP)ส่งข้อมูลเข้าระบบ SAP B1 แล้ว
Received (ได้รับ)SAP ยืนยันแล้ว แสดงเลขเอกสารจริง

วิธีใช้งาน

  • คลิก + New PR หรือ + New PO เพื่อสร้างเอกสาร
  • กรอก: โปรเจค, ผู้ขาย, รายการสินค้า (สินค้า, จำนวน, ราคา)
  • สลับมุมมองระหว่าง Kanban และ List
  • คลิกเลขเอกสารเพื่อดูรายละเอียดและ Document Flow
  • สถานะเอกสารจะเปลี่ยนตามขั้นตอนอนุมัติ ไม่สามารถลากย้ายเองได้
พอ SAP รับเอกสารแล้ว จะได้เลขที่จริงจาก SAP กลับมาโชว์เป็น badge สีฟ้าในระบบ
เงินทดรองจ่าย (Advance) เอกสาร ▼

ภาพรวม

พนักงานจะขอเบิกเงินล่วงหน้าไปใช้จ่าย พอใช้เสร็จก็มาเคลียร์กับใบเสร็จ ถ้าเงินเหลือก็คืน ถ้าไม่พอบริษัทจ่ายเพิ่มให้

ขั้นตอนการทำงาน

ขอเบิก→ อนุมัติ→ จ่ายเงิน→ เคลียร์→ ปิด→ พนักงานยืนยัน

รายละเอียดแต่ละขั้นตอน

ขั้นตอนผู้ดำเนินการสิ่งที่ต้องทำ
1. ขอเบิกพนักงานกรอกว่าจะเบิกเท่าไหร่ ไปทำอะไร
2. อนุมัติหัวหน้า / การเงินเช็คแล้วกดอนุมัติ (เงินเยอะต้องผ่านหลายคน)
3. จ่ายเงินการเงินโอนเงินหรือจ่ายสดให้พนักงาน แนบสลิปด้วย
4. เคลียร์พนักงานใช้เงินเสร็จแล้วเอาใบเสร็จมาเคลียร์
5. ตรวจสอบการเงินเช็คใบเสร็จ กดอนุมัติ ถ้ามีส่วนต่างก็จัดการคืน/จ่ายเพิ่ม
6. ยืนยันพนักงานกดยืนยันว่าได้รับเงินเพิ่มแล้ว (เฉพาะกรณีบริษัทจ่ายเพิ่ม)

ระดับการอนุมัติ

จำนวนเงินต้องได้รับอนุมัติจาก
ไม่เกิน 10,000 บาทผู้จัดการเท่านั้น
10,001 - 100,000 บาทผู้จัดการ + การเงิน
มากกว่า 100,000 บาทผู้จัดการ + การเงิน + ผู้บริหาร

ภาพรวม

เงินสดย่อยที่เก็บไว้ในออฟฟิศสำหรับจ่ายค่าเบ็ดเตล็ดต่างๆ เช่น ค่าแท็กซี่ ค่ากาแฟ ค่าพัสดุ ระบบจะคอยดูแลยอดเงิน บัญชี และเตือนเมื่อเงินใกล้หมด

ขั้นตอนการทำงาน

สร้างกองทุน→ เบิกจ่าย→ ส่งใบเสร็จ→ อนุมัติ→ เติมเงิน→ นับเงินปิดงวด

รายละเอียดแต่ละขั้นตอน

ขั้นตอนผู้ดำเนินการสิ่งที่ต้องทำ
1. สร้างกองทุนAdmin / การเงินตั้งชื่อ กำหนดวงเงิน เลือกคนดูแล
2. เบิกจ่ายคนดูแลจ่ายเงินสดให้พนักงาน ลงบันทึกว่าจ่ายใคร เท่าไหร่ ซื้ออะไร
3. ส่งใบเสร็จพนักงานเอาใบเสร็จมาส่ง ใส่ยอดจริง ถ้ามีเงินทอนระบบคำนวณให้
4. อนุมัติใบเสร็จคนดูแล / การเงินเช็คใบเสร็จแล้วกดอนุมัติ ระบบลงบัญชีให้เอง
5. เติมเงินคนดูแลเงินใกล้หมดก็กดขอเติม การเงินอนุมัติแล้วโอนเงินเข้ามา
6. นับเงินปิดงวดคนดูแลนับเงินสดในกล่อง ใส่ยอดจริง ระบบเทียบให้ว่าตรงไหม

ภาพรวม

ดูรายชื่อคนที่ใช้ระบบ กำหนดว่าใครมีสิทธิ์ทำอะไรได้บ้าง อยู่โปรเจคไหน

บทบาทผู้ใช้งาน

บทบาทสิทธิ์การเข้าถึง
Executive (ผู้บริหาร)เห็นทุกอย่าง อนุมัติได้ทุกระดับ
Admin (แอดมิน)ทำได้ทุกอย่าง จัดการคนและตั้งค่าระบบ
PM (ผู้จัดการโปรเจค)ดูแลโปรเจคที่รับผิดชอบ อนุมัติงานของลูกทีม
Finance (การเงิน)จ่ายเงิน อนุมัติเบิกจ่าย ดูแลงบ
Accounting (บัญชี)ดูรายการบัญชี GL จัดการค่าใช้จ่าย
Procurement (จัดซื้อ)สร้าง PR/PO ดูแลเรื่องผู้ขายและสินค้า
Staff (พนักงานทั่วไป)ดูโปรเจคตัวเอง ส่งคำขอต่างๆ ได้

วิธีใช้งาน

  • ดูรายชื่อผู้ใช้งานและบทบาท
  • คลิกที่ผู้ใช้เพื่อแก้ไขบทบาท โปรเจค และสถานะ
  • มอบหมายผู้ใช้ให้กับโปรเจคเฉพาะ
  • สมัครสมาชิก — ผู้ใช้ใหม่สามารถสมัครได้ที่หน้า /register.html
