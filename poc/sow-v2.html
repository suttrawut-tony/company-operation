3 บทสรุปผู้บริหาร (Executive Summary)

Company Operation v2.0 เป็น SaaS Multi-tenant Operation Management Platform ที่พัฒนาขึ้นเองบน Modern Web Stack (Express.js + PostgreSQL + Static Frontend) เพื่อรวมศูนย์การบริหารงานปฏิบัติการขององค์กรไว้ในแพลตฟอร์มเดียว ครอบคลุมตั้งแต่การบริหารโครงการ (Project & Task Management), กระบวนการจัดซื้อ (Procurement), การเงินและบัญชี (Financial), ทรัพยากรบุคคล (HR & Admin), ไปจนถึงการจองทรัพยากร (Resource Booking) และเชื่อมต่อระบบ SAP Business One ผ่าน Service Layer API

ระบบรองรับ Multi-company ด้วยการแยกข้อมูลระดับ company_id พร้อมระบบ Subscription 3 ระดับ (Starter / Professional / Enterprise) และ Trial 14 วัน ระบบมี RBAC 4 ระดับ, Approval Workflow แบบ Configurable, Real-time Notification ผ่าน WebSocket รวมถึงมาตรการ Security ตามมาตรฐาน (Helmet.js, Rate Limiting, CSP, JWT)

SOW ฉบับนี้กำหนดขอบเขตสำหรับการพัฒนา ทดสอบ และ Deploy ระบบทั้ง 37 modules จัดเป็น 10 กลุ่ม โดยเป็นโครงการพัฒนาซอฟต์แวร์ (Custom SaaS Development) ไม่ใช่การ Implement ระบบ ERP สำเร็จรูป

10 สมมติฐานโครงการ (Assumptions)

  1. ประเภทโครงการ: พัฒนา SaaS Web App ไม่ใช่ ERP Implementation -- ใช้ Software Requirement Spec ไม่ใช่ ERP Blueprint
  2. Hosting: Deploy บน Railway (Cloud PaaS) -- ลูกค้าไม่ต้องจัดเตรียม server
  3. Browser-based: ทำงานผ่าน Web Browser ตาม Section 6.4 ไม่รองรับ Mobile Native
  4. Multi-tenancy: แยกด้วย company_id (row-level) ไม่ใช่ database-level -- dedicated DB ต้องตกลงแยก
  5. SAP B1 Connection: ลูกค้าจัดเตรียม SAP credentials + network access -- ผู้ให้บริการไม่รับผิดชอบ SAP License/uptime
  6. Data Ownership: ข้อมูลเป็นทรัพย์สินของลูกค้า -- ผู้ให้บริการไม่เข้าถึงโดยไม่ได้รับอนุญาต
  7. Feedback: ลูกค้าให้ feedback ภายใน {{N}} วันทำงาน ล่าช้าอาจกระทบ timeline
  8. Scope Freeze: หลัง P1 อนุมัติ requirement ถือเป็น scope ตกลง -- เปลี่ยนผ่าน Change Request
  9. Concurrent Users: รองรับ {{N}} concurrent users/tenant -- เพิ่มต้องประเมิน infra cost
  10. Payment: ชำระตามงวดใน Section 13 ก่อนเริ่มงานแต่ละ Phase

  2. ห้ามเปิดเผยต่อบุคคลที่สามโดยไม่ได้รับอนุญาต ยกเว้นพนักงาน/ผู้รับจ้างที่จำเป็นต้องทราบ
  3. ข้อผูกพันมีผลตลอดโครงการและอีก {{N}} ปีหลังสิ้นสุด
  4. ข้อยกเว้น: ข้อมูลสาธารณะ, ข้อมูลที่มีอยู่ก่อน, คำสั่งศาล/หน่วยงานรัฐ

16 การคุ้มครองข้อมูลส่วนบุคคล (PDPA Compliance)

เพื่อให้เป็นไปตาม พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 (PDPA):

รายการรายละเอียด
Data Controllerลูกค้า -- ผู้ควบคุมข้อมูลของพนักงาน คู่ค้า ผู้ใช้งาน

17 การยกเลิกสัญญา (Termination)

17.1 ยกเลิกโดยแจ้งล่วงหน้า

ฝ่ายใดฝ่ายหนึ่งแจ้งเป็นลายลักษณ์อักษรล่วงหน้าไม่น้อยกว่า {{N}} วัน

17.2 ยกเลิกเนื่องจากผิดสัญญา

หากฝ่ายใด material breach และไม่แก้ไขภายใน {{N}} วันทำงานหลังได้รับแจ้ง อีกฝ่ายยกเลิกได้ทันที
